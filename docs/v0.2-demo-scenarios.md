# v0.2 Demo Scenarios

## Mock scenario 1: baseline

Purpose:

- show a healthy fleet with complete collections
- review `Targets`, `Risks`, `Findings`, and `Evidence` without degraded noise

Run:

```bash
SIDECAR_MOCK_SCENARIO=baseline corepack pnpm dev
```

Suggested walkthrough:

1. Open `Overview`
2. Open `Targets`
3. Open `Risks`
4. Drill into a finding and then into evidence

## Mock scenario 2: partial-coverage

Purpose:

- demonstrate partial and unavailable collections
- verify coverage gaps and recommendations remain readable

Run:

```bash
SIDECAR_MOCK_SCENARIO=partial-coverage corepack pnpm dev
```

Suggested walkthrough:

1. Open `Targets` and review coverage badges
2. Open `Evidence` and filter by `coverage-gap`
3. Open `Findings` and review workspace/config issues

## Filesystem fixture scenario

Purpose:

- show local-path integration and degraded warnings when files are incomplete
- validate governance findings against a realistic runtime fixture

Verification path used in tests:

- `test/filesystem-adapter.test.ts`
- `test/governance-engine.test.ts`
- `test/overlay-api.readonly.test.ts`

Suggested operator flow:

1. set `OPENCLAW_STATE_DIR` or `OPENCLAW_RUNTIME_ROOT`
2. optionally set `OPENCLAW_CONFIG_PATH`
3. optionally set `OPENCLAW_PROFILE`
4. set `OPENCLAW_WORKSPACE_GLOB` only if your workspace layout is not covered by the official `~/.openclaw/workspace*` default
5. start `corepack pnpm dev`
6. open `Targets` to confirm source metadata
7. open `Risks` and `Evidence` to review degraded signals
