# v0.2 Known Limitations

## Product limits

- read-only only
- no chat UX
- no execution control
- no write-back or suppress/ack state
- no RBAC

## Source limits

- `filesystem` is the only current real adapter
- gateway, dashboard, CLI, and log collectors are still planned, not implemented
- multi-target support depends on local target registry configuration, not service discovery

## Validation limits

- only one minimal browser-level E2E is currently in place
- screenshot artifacts are not yet checked into the repository
- sample API examples are illustrative and not a substitute for live validation
