# v0.3 Changelog

## Added

- `Target` registry model and target-scoped summary contracts
- multi-target support via `SIDECAR_TARGETS_FILE`
- governance models:
  - `Evidence`
  - `Finding`
  - `Recommendation`
  - `RisksSummary`
- new overlay-api endpoints:
  - `/api/targets`
  - `/api/targets/:id`
  - `/api/targets/:id/summary`
  - `/api/evidence`
  - `/api/evidence/:id`
  - `/api/findings`
  - `/api/findings/:id`
  - `/api/recommendations`
  - `/api/recommendations/:id`
  - `/api/risks/summary`
- new overlay-web views:
  - `Targets`
  - `Target Detail`
  - `Risks`
  - `Findings`
  - `Finding Detail`
  - `Evidence`
  - `Evidence Detail`
- recommendation rendering in `Finding Detail`
- target-to-findings and target-to-evidence jump links

## Changed

- product positioning shifted from control wording to governance and visibility wording
- overview now includes fleet-level target and risk signals
- README and local deployment docs now describe target registry and governance flows
- `.env.example` now documents target registry and source metadata variables

## Preserved

- standalone repository boundary
- `sidecar + overlay-api + overlay-web` architecture
- GET-only overlay-api behavior
- strict read-only contract
- mock-first default startup
- filesystem as the only current real adapter

## Not included

- write-back workflows
- remote execution
- chat UX
- RBAC
- real-time event bus
