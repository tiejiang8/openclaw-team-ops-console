# v0.3 DTO Changes

## New models

- `Target`
- `TargetSnapshotSummary`
- `Evidence`
- `Finding`
- `Recommendation`
- `RisksSummary`
- `RuntimeStatusDto`
- `CronJobSummaryDto`
- `CronJobDetailDto`
- `NodeSummaryDto`

## Extended models

- `TargetSnapshotSummary` now carries target-scoped inventory arrays
- response contracts now include target, evidence, finding, recommendation, and risks-summary responses
- runtime contracts now include read-only gateway / OpenClaw / cron / nodes runtime plane state
- `NodesResponse` now carries a top-level summary for paired / connected / stale counts

## Purpose

These changes allow the console to represent governance objects explicitly instead of inferring them from loose warning fields, while also exposing runtime-plane observability through shared contracts.
