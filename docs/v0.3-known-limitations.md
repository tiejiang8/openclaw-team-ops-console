# v0.3 Known Limitations

## Product limits

- read-only only
- no chat UX
- no execution control
- no write-back or suppress/ack state
- no RBAC

## Source limits

- `gateway-ws` is wired only for `operator.read` runtime-plane calls
- write RPCs remain explicitly out of scope
- runtime events currently refresh the sidecar cache, but the web still renders via polling rather than SSE
- multi-target support depends on local target registry configuration, not service discovery

## Validation limits

- browser E2E now covers runtime status, cron, and nodes views, but screenshot artifacts are still not checked in
- sample API examples are illustrative and not a substitute for live validation
